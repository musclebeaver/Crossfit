# Crossfit Platform - 테스트 시나리오 목록

이 문서는 프론트엔드(Flutter)와 백엔드(Spring Boot) 간의 연동 및 전반적인 기능 완성도를 검증하기 위한 수동 테스트 시나리오입니다.

## 1. 인증 및 권한 (Authentication)
- [ ] **회원가입**: 이메일, 패스워드, 닉네임 입력 시 정상적으로 계정이 생성되는가? (중복 이메일/닉네임 거절 확인)
- [ ] **일반 로그인**: 가입한 이메일과 비밀번호(또는 더미: `user1@test.com` / `1234`)로 로그인 시 홈 화면으로 진입하는가?
- [ ] **소셜 로그인 (OAuth2)**: 구글, 카카오, 네이버 버튼 클릭 시 정상적으로 소셜 인증 후 JWT 토큰을 발급받아 홈으로 리다이렉트 되는가?
- [ ] **로그아웃**: 마이페이지에서 로그아웃 시 토큰이 파기되고 로그인 화면으로 전환되는가?
- [ ] **어드민 로그인**: 더미 어드민(`admin1@test.com` / `1234`) 로그인 시 일반 화면이 아닌 'Admin Console' 화면으로 진입하는가?

## 2. 박스(Box) 탐색 및 가입
- [ ] **박스 검색**: 'My Box' 탭에서 검색창에 'Tiger' 혹은 'Beaver' 입력 시 관련 박스가 목록에 노출되는가?
- [ ] **박스 가입 신청**: 특정 박스의 'Join' 버튼 클릭 시 가입 신청(Pending 상태)이 전송되는가?
- [ ] **내 상태 확인**: 가입 신청 직후 'My Box' 화면에 내 상태(Status: PENDING)가 정상적으로 표시되는가?
- [ ] **멤버 승인 (Admin/Coach)**: 박스의 소유자(Coach) 계정으로 로그인 후, 멤버 관리 탭에서 유저의 가입 신청을 Approve/Reject 할 수 있는가?
- [ ] **승인 후 화면 변화**: 승인(Approve) 처리된 유저가 다시 접속했을 때, My Status가 초록색 체크 모양(APPROVED)으로 변경되어 있는가?

## 3. 기록 및 마이페이지 (My Page & Records)
- [ ] **내 정보 조회**: 프로필 탭 진입 시 본인의 닉네임, 티어(Tier), 총 포인트가 정확히 렌더링되는가?
- [ ] **닉네임 변경**: Change Nickname 클릭 후 새로운 닉네임을 입력/저장하면 즉시 화면에 반영되는가?
- [ ] **비밀번호 변경**: 기존 비밀번호(`1234`)를 입력하고 새 비밀번호를 입력 시 성공적으로 변경되는가?
- [ ] **이전 WOD 기록 차트 조회**: Records 탭 진입 시 최상단에 본인의 랭킹 히스토리가 꺾은선 그래프(FlChart)로 나타나는가?
- [ ] **이전 WOD 기록 상세 리스트**: 그래프 하단에 참여했던 WOD 리스트와 순위, Rx'd/Scaled 여부가 노출되는가?

## 4. WOD 및 랭킹 시스템
- [ ] **오늘의 WOD 조회**: 홈 파트(또는 WOD 리스트 화면)에서 오늘의 글로벌 WOD 및 내 박스의 WOD가 노출되는가?
- [ ] **WOD 기록 등록**: 특정 WOD 선택 후 기록(reps/time)과 방식(Rx'd/Scaled)을 입력해 제출하면 데이터베이스에 저장되는가?
- [ ] **실시간 랭킹 (Redis)**: 기록 등록 직후 Ranking 탭에 즉각적으로 본인의 닉네임과 순위가 노출되는가?
- [ ] **랭킹 타임캡 처리 로직**: Time Capped에 체크한 유저가 끝까지 완주한 기기(동일 스케일)보다 무조건 하위에 랭크되는가?
- [ ] **스케일 로직 처리**: Rx'd 버튼을 누른 사용자가 Scaled 버튼을 누른 사용자보다 무조건 상위에 랭크되는가?

## 5. 어드민 뷰 및 AI WOD (Admin Console)
- [ ] **박스 리스트 조회**: 어드민 계정 로그인 > Boxes 탭에서 10개의 박스 더미 데이터가 노출되는가?
- [ ] **유저 리스트 조회**: Users 탭에서 총 30명+알파의 유저 데이터와 각 티어, 역할(Role) 배지가 렌더링되는가?
- [ ] **글로벌 WOD 캘린더**: AI WOD 탭에서 캘린더의 날짜를 누를 때 해당 날짜에 배정된 WOD 목록이 하단에 출력되는가?
- [ ] **AI WOD 자동 생성 다이얼로그**: 박스 관리 화면이나 글로벌 생성 화면에서 'AI Auto Generate' 버튼을 누른 뒤, Team Size 및 Team Format 선택 폼이 정상적으로 작동하는가?
- [ ] **AI WOD 생성 응답 확인**: 요구사항 입력 후 Generate 버튼 시 Gemini API를 거쳐 WOD 구조가 파싱되어 목록에 새롭게 추가되는가?

## 6. 스케줄러 및 티어 정산 (Redis & Scheduler)
- [ ] **어제 날짜 기록 초기화 확인**: 앱 빌드 직후('DummyDataInit' 구동), DB상에 어제(`yesterday`) 날짜를 가진 글로벌 WOD와 유저들의 랭킹보드용 더미 Record가 잘 저장되었는가?
- [ ] **포인트 정산 API 트리거**: 최고관리자(Admin) 권한 토큰을 발급받아 Postman 등을 이용해 `POST /api/v1/admin/trigger-ranking-scheduler`를 빈 바디로 호출했을 때 응답이 잘 발생하는가?
- [ ] **랭킹 히스토리 마이그레이션**: 호출 성공 후 즉시 DB의 `RankingHistory` 테이블을 조회해 보았을 때, Redis에 임시 저장되었던 30건의 유저들 등수/점수/Rx여부가 DB로 이관되는가?
- [ ] **유저 티어 차등 승급 확인**: `User` 테이블을 조회했을 때 백분율과 순위에 따라 지급된 Points가 정상적으로 더해져, 티어가 `AMATEUR`, `PRO` 등으로 차등 승급되었는지 확인되는가?
