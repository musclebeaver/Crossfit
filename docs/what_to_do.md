# 향후 개발 로드맵 (What to Do)

앱 출시 및 서비스 안정화를 위해 향후 진행해야 할 핵심 작업 목록입니다.

## 1. 결제 시스템 실연동 (Monetization)
- [ ] **In-App Purchase (IAP) 연동**: 현재 모의 결제로 구현된 '프리미엄 업그레이드'를 `in_app_purchase` 라이브러리를 통해 Apple App Store 및 Google Play Billing 실연동으로 전환.
- [ ] **정기 구독(Subscription) 모델**: 단발성 포인트 구매 외에 월간 멤버십(광고 제거 + AI 무제한) 구독 로직 백엔드 연동.

## 2. 알림 및 리텐션 고도화 (Retention)
- [ ] **FCM 실제 인증서 등록**: Firebase Console에서 APNs(Apple) 및 Android용 실제 서버 키/인증서를 등록하여 실제 기기에서 푸시 알림 확인.
- [ ] **알림 히스토리 화면**: 사용자가 받은 푸시 알림 목록을 앱 내에서 모아볼 수 있는 탭/화면 추가.

## 3. 운영자 관리 도구 (Admin Console)
- [ ] **신고 관리 UI**: `reports` 테이블에 쌓인 신고 내역을 확인하고, 부적절한 유저의 계정을 정지하거나 기록을 삭제할 수 있는 어드민 대시보드 구축.
- [ ] **박스 승인 관리**: 코치 가입 신청 시 시스템 관리자가 박스의 진위 여부를 확인하고 승인해주는 슈퍼 어드민 로직.

## 4. 인프라 및 배포 (DevOps)
- [ ] **프로덕션 환경 구축**: `.env` 파일의 소셜 로그인 Client ID, Secret 등을 테스트용이 아닌 실제 정식 발급된 키로 교체.
- [ ] **배포 자동화 (CI/CD)**: Fastlane 또는 GitHub Actions를 연동하여 코드 푸시 시 자동으로 빌드 및 스토어(테스트플라이트 등) 배포 환경 구축.
- [ ] **도메인 및 SSL**: 서버 IP 주소가 아닌 정식 도메인 연동 및 HTTPS 보안 인증서(SSL) 적용.

## 5. 지표 분석 및 마케팅 (Analytics)
- [ ] **Firebase Analytics 연동**: 유저의 이탈 구간, 인기 WOD 종류 등 행동 지표를 분석하기 위한 이벤트 로깅 추가.
- [ ] **Crashlytics**: 앱 비정상 종료 시 원인을 파악하기 위한 크래시 리포팅 도구 연동.

## 6. 스토어 등록 정보 준비
- [ ] **스크린샷 및 동영상**: 마케팅용 고해상도 앱 스크린샷 5장 내외 제작.
- [ ] **데이터 보안 섹션 작성**: 구글/애플 개발자 센터의 데이터 수집 항목(이메일, 기기 ID 등) 선언문 작성.
- [ ] **콘텐츠 등급 설문**: 앱 스토어 등급 지정을 위한 설문 작성.
